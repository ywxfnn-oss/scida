# Scida v1.6.0

## English

### What this release focused on

v1.6.0 upgraded the database page into a stronger personal research workspace for returning to older records, organizing important experiments, and moving into read-only analysis more efficiently.

### Highlights

- saved local filter views for database browse state
- star / unstar from list and detail, with persistence across restart
- starred-only quick filtering
- related records on detail using same-project, same-sample, and same-tester relationships
- direct `open selected in analysis` navigation from the database list

### What stayed intentionally narrow

- no schema change
- no authored record-relationship system
- no analysis editing
- no export redesign
- no cloud or collaboration expansion

### Why it matters

Scida’s local record history now feels more like a usable personal knowledge base instead of only a long list that users browse passively.

## 中文

### 这个版本重点做了什么

v1.6.0 把数据库页进一步升级成更像“个人科研工作台”的入口，重点改善回看旧实验、组织重要记录，以及更高效进入只读分析的体验。

### 主要亮点

- 支持保存数据库筛选状态为本地常用视图
- 支持在列表和详情页对记录进行星标 / 取消星标，并在重启后保留
- 支持仅查看星标记录
- 详情页新增相关记录导航，基于同项目、同样品、同测试人派生
- 数据库列表新增 `在分析中打开所选记录` 的快捷入口

### 有意识保持收敛的范围

- 不改 schema
- 不引入用户手工维护的记录关系系统
- 不把分析变成编辑器
- 不重做导出
- 不扩展到云端或协作

### 为什么它重要

Scida 的历史记录不再只是一个被动浏览的长列表，而开始更像一个真正可返回、可组织、可复用的个人研究知识库入口。

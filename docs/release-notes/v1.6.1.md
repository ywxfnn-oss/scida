# Scida v1.6.1

## English

### What this release focused on

v1.6.1 is a focused polish release across the workflow surfaces added after v1.4. It keeps the product boundaries unchanged while making database, entry, structured-data, and analysis review behavior more consistent and easier to read.

### Highlights

- clearer database workspace hierarchy and interaction grouping
- custom 24-hour test-time picker that does not depend on the system AM/PM preference
- simplified one-block-one-curve structured-data import flow
- more compact structured-data review in database detail
- consistent structured-series naming between Step 2, database detail, and Analysis

### Important fixes and refinements

- stronger completeness guidance around partially filled rows
- safer draft clearing after final save
- leave-flow handling that respects draft workflow
- clearer copied-draft status wording

### Why it matters

This release reduces the remaining “rough edges” that could make Scida feel inconsistent after the larger v1.5 and v1.6 workflow upgrades.

## 中文

### 这个版本重点做了什么

v1.6.1 是一个聚焦打磨的一致性版本，主要收敛 v1.4 之后新增工作流里的界面、命名和交互边缘问题。它不改变产品边界，但让数据库、录入、结构化数据和分析回看这几条链路更加一致、可读、可用。

### 主要亮点

- 数据库工作台的信息层级和操作分组更清楚
- 自定义 24 小时制测试时间选择器，不再受系统 AM/PM 偏好影响
- 结构化数据导入回到“一块一条 X/Y 曲线”的更直接流程
- 数据库详情页中的结构化数据回看更紧凑
- Step 2、数据库详情、Analysis 三处的结构化曲线命名更一致

### 重要修复与细化

- 更准确地提示部分填写但未完成的条目
- 最终保存后草稿清理边界更安全
- 离开录入流程时会正确走草稿处理链路
- 复制草稿后的状态提示文案更完整

### 为什么它重要

这个版本主要消除 v1.5 和 v1.6 两轮升级之后仍残留的不一致和粗糙感，让 Scida 的主工作流更连贯。

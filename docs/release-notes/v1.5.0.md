# Scida v1.5.0

## English

### What this release focused on

v1.5.0 made Scida better for repeated day-to-day experiment entry. The product already had the core create-save workflow, but repeated similar entry and unfinished-record handling still had too much friction.

### Highlights

- one active local draft with:
  - `Save Draft and Return`
  - `Resume Draft`
  - `Discard Draft`
- `Copy as New Draft` from database detail for fast reuse of existing record context
- non-blocking completeness guidance for missing Step 1 fields, pending dictionary confirmation, and incomplete Step 2 content
- recent suggestions for commonly reused `testProject` and `instrument`
- faster post-save loop with:
  - `Create Similar`
  - `Open Saved Record`
  - `Back Home`

### Boundaries kept intact

- analysis remains read-only
- export behavior remains unchanged
- drafts do not appear in database browse, analysis, or export
- the release stays local-first and does not add cloud, collaboration, or AI features

### Why it matters

This release makes Scida more usable as an everyday recording tool rather than only a record archive that users return to occasionally.

## 中文

### 这个版本重点做了什么

v1.5.0 重点提升了高频、重复性的日常实验录入体验。Scida 原本已经具备核心的录入与保存流程，但对于“相似记录反复录入”和“中途没录完的记录”仍然存在明显摩擦。

### 主要亮点

- 单个本地活动草稿能力：
  - `保存草稿并返回`
  - `继续草稿`
  - `丢弃草稿`
- 从数据库详情页发起 `复制为新草稿`，更快复用已有记录上下文
- 非阻塞的完整性提示，可提示：
  - Step 1 必填字段缺失
  - 字典项仍待确认
  - Step 2 尚无内容
  - 标量项或结构化项填写不完整
- 针对常用 `测试项目` 和 `测试仪器` 的最近使用建议
- 完成保存后的更顺滑回路：
  - `创建相似记录`
  - `打开已保存记录`
  - `返回首页`

### 保持不变的边界

- 分析仍然保持只读
- 导出行为不变
- 草稿不会进入数据库浏览、分析或导出
- 仍然坚持本地优先，不引入云端、协作或 AI 功能

### 为什么它重要

这个版本让 Scida 更像一个可以每天持续使用的录入工具，而不只是一个事后归档和回看的记录容器。

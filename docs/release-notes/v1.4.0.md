# Scida v1.4.0

## English

### What Scida is

Scida is a local-first scientific data workspace for individual experimental research workflows. It keeps experiment records, raw files, structured data, lightweight analysis, and export in one desktop workflow.

### Why this release matters

v1.4.0 is an important public-facing release because Scida is now much easier to understand and use for both Chinese-speaking and international users. The product shell and main guided workflows are substantially more bilingual than earlier versions.

### Highlights

- bilingual `zh-CN` / `en` UI support with persisted language preference
- broader localization across login, onboarding, Step 1, Step 2, database workflows, and analysis guidance
- improved About / release shell polish
- cleaner Scida branding and icon integration
- continued local-first workflow centered on records, files, analysis, and export

### Current domain focus

The product is still shaped most directly by photodetector and optoelectronic experiment workflows, because that is the real research background it was built from.

### Current limitations

- Scida is still a personal research tool, not a full LIMS or team ELN
- some deeper workflow wording may continue to evolve
- public binaries should still be treated conservatively unless a release explicitly adds signing/notarization

### Feedback welcome

If Scida overlaps with your own research workflow, feedback is very welcome, especially around templates, record structure, export needs, bilingual clarity, and workflow friction.

## 中文

### Scida 是什么

Scida 是一个面向个人实验科研工作流的、本地优先 scientific data workspace，用来把实验记录、原始文件、结构化数据、轻量分析和导出整合进同一个桌面工作流。

### 为什么这个版本重要

v1.4.0 是一个很重要的公开版本，因为 Scida 现在对中文用户和国际用户都更容易理解和使用了。相比更早的版本，产品外壳和主要引导式工作流已经有了更完整的双语支持。

### 主要亮点

- 支持持久化保存的 `zh-CN` / `en` 双语界面
- 登录、onboarding、Step 1、Step 2、数据库工作流和分析引导文本都有了更完整的本地化
- About / 发布外壳进一步完善
- Scida 品牌名和图标整合更完整
- 继续坚持以记录、文件、分析和导出为核心的本地优先工作流

### 当前领域侧重点

这个产品目前仍然最直接地围绕光电探测器和光电子实验工作流展开，因为它本来就是从这个真实研究背景中长出来的。

### 当前限制

- Scida 仍然是一个个人科研工具，不是完整的 LIMS 或团队 ELN
- 一些更深层的工作流文案后续仍会继续调整
- 在后续 release 明确加入签名/notarization 之前，公开二进制仍应保守看待

### 欢迎反馈

如果 Scida 和你的科研工作流有重叠，非常欢迎反馈，尤其是模板设计、记录结构、导出需求、双语表达清晰度，以及使用过程中感受到的流程摩擦。
